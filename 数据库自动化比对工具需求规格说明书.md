# 1. 文档概述

## 1.1 文档目的

本文档为数据库自动化比对工具的正式需求规格说明，明确工具的开发目标、核心功能、非功能要求、业务场景、验收标准等全部核心需求，用于指导工具的设计、开发、测试、部署及后期迭代优化，同时作为需求方（软件测试团队）与开发方的沟通依据和验收基准，确保工具完全解决ETL数据抽取后的源目数据库人工比对痛点，实现自动化、精准、高效的结构与数据一致性校验。

## 1.2 文档范围

本文档覆盖工具的全部业务需求、功能需求、非功能需求、界面交互、环境部署、验收规则及风险备注，适用于公司内部ETL业务场景下的源数据库与目标数据库全量/增量结构比对、数据比对工作，仅限内部测试、运维、开发人员使用，不对外提供服务。

## 1.3 术语定义

- **源数据库**：ETL抽取流程中，提供原始表结构和业务数据的数据库，为数据比对的基准库

- **目标数据库**：ETL抽取流程中，接收并存储抽取后表结构和数据的数据库，为数据比对的校验库

- **结构比对**：针对源、目标库中表的存在性、字段属性、约束、索引等元数据信息的一致性校验

- **数据比对**：针对源、目标库中对应表内实际业务数据的行数、字段值、唯一键值等内容的一致性校验

- **ETL**：数据抽取、转换、加载的业务流程，为本工具的核心应用场景

- **全量比对**：对指定库内全部表/单表全部数据进行完整比对

- **增量比对**：仅对指定时间范围、指定批次的新增/更新数据进行比对，提升效率

# 2. 项目背景与核心目标

## 2.1 项目背景

当前团队核心业务为ETL数据抽取，需从各类源数据库抽取表结构及对应业务数据，同步至指定目标数据库，完成数据迁移与归集。目前校验环节完全依赖人工操作：测试人员需手动连接源库和目标库，逐表查看表结构定义、逐行核对数据内容，确认抽取后的结构与数据和原始数据完全一致。

该人工模式存在诸多痛点：一是耗时极长，单批次几十张表的比对工作需耗费数小时，大批量表比对甚至需要1-2天，严重占用测试人力和项目时间；二是易错率高，人工肉眼核对易出现漏查、错查，尤其数值型、字符型细微差异难以识别；三是无法批量处理，仅支持单表逐次比对，无统一的差异汇总和报告输出；四是无追溯性，人工核对无记录，后期问题排查无依据，不符合软件测试的规范化要求。

## 2.2 核心目标

1. 替代人工操作，实现源、目标数据库表结构和数据的自动化比对，大幅降低人力成本，将比对效率提升80%以上

2. 精准识别所有结构差异、数据差异，定位差异位置和具体内容，无漏判、无错判

3. 支持多数据库类型适配，满足团队现有ETL业务的数据库环境需求

4. 输出标准化、可导出的比对报告，方便测试人员存档、问题反馈和追溯

5. 操作简单易上手，无需复杂配置，适配测试人员日常工作习惯，降低使用门槛

# 3. 用户角色与应用场景

## 3.1 用户角色

- **软件测试人员（核心用户）**：负责ETL流程后的校验工作，使用工具完成结构、数据比对，出具测试结论，定位数据抽取缺陷

- **ETL开发人员**：使用工具快速定位抽取逻辑问题，优化ETL脚本，修复结构和数据差异

- **数据库运维人员**：使用工具核查数据迁移完整性，排查数据库同步异常

## 3.2 核心应用场景

1. **ETL流程上线前校验**：新开发的ETL脚本部署前，通过工具比对源目数据一致性，验证抽取逻辑正确性

2. **日常回归测试校验**：ETL脚本迭代优化后，批量比对相关表，确认无新增差异，回归原有功能

3. **批量数据迁移校验**：大批量表同步完成后，全量比对结构和数据，确保迁移无遗漏、无错误

4. **线上问题排查**：业务系统出现数据异常时，快速比对源目数据，定位是ETL抽取问题还是数据库本身问题

5. **增量数据校验**：定时增量抽取后，仅核对新增/更新数据，无需全量比对，提升校验效率

# 4. 核心功能需求

## 4.1 数据源配置管理

本模块为工具基础功能，支持用户灵活配置源数据库和目标数据库连接信息，实现多数据源管理，避免重复配置。

- 支持主流数据库类型适配：MySQL、Oracle、SQL Server、PostgreSQL、达梦（DM）、Inceptor（覆盖团队现有ETL业务用到的数据库）

- 支持数据库连接信息配置：数据库地址、端口、服务名/库名、Schema名、用户名、密码、连接超时时间、字符集编码

- 支持连接测试功能：配置完成后一键测试连通性，提示连接成功/失败及失败原因（如密码错误、网络不通、权限不足）

- 支持数据源保存与编辑：保存常用的源库、目标库配置，支持修改、删除、重命名配置，无需每次重新录入

- 支持单组/多组数据源切换：快速切换已保存的数据源，适配不同业务线的ETL比对需求

- 连接信息安全存储：对数据库密码进行加密存储，禁止明文展示，保障数据库信息安全

- 支持数据源分组管理：按业务线、项目等维度对数据源进行分组，方便管理和查找

## 4.2 表结构自动化比对功能

本模块为核心功能之一，针对源、目标库的表元数据进行全维度比对，覆盖ETL抽取后结构一致性的全部校验点，精准识别各类结构差异。

### 4.2.1 比对范围配置

- 支持全库表比对：自动加载源库所有表，批量比对目标库对应表的结构

- 支持指定表比对：手动输入表名、批量导入表名列表，仅比对指定的单张/多张表（核心测试场景常用）

- 支持排除表配置：全库比对时，可排除系统表、临时表等无需比对的表

- 支持表名映射配置：源表与目标表名称不同时，支持配置表名映射关系（如：源表SRC_USER对应目标表TGT_USER）

- 支持Schema映射配置：源库与目标库Schema不同时，支持配置Schema映射关系

### 4.2.2 结构比对项（必选，不可省略）

- 表存在性比对：校验源库存在的表，目标库是否存在；目标库多余的表是否为ETL误生成

- 字段基础信息比对：字段名称、字段数据类型、字段长度、字段精度、是否为空（NULL）属性

- 字段约束比对：主键约束、唯一约束、非空约束、默认值

- 索引比对：主键索引、普通索引、唯一索引的名称、关联字段、索引类型

- 表注释、字段注释比对：校验表说明、字段说明是否同步完整

- 字段映射比对：支持源表与目标表字段名称不同时的映射比对（ETL过程中可能存在字段重命名）

### 4.2.3 结构差异结果展示

- 分类展示差异：分为源库有目标库无、目标库有源库无、字段属性不一致、约束不一致、索引不一致、注释不一致六大类

- 明细展示差异内容：明确标注表名、差异字段、源库属性值、目标库属性值、差异类型

- 支持结构差异筛选：按差异类型、表名快速筛选定位问题表

## 4.3 数据自动化比对功能

本模块为核心功能之二，针对源、目标库对应表的实际业务数据进行一致性比对，支持全量和增量两种模式，解决人工核对数据易错、耗时的核心痛点。

### 4.3.1 比对模式

- **全量数据比对**：对指定表内所有数据行进行逐行、逐字段比对，适用于首次迁移、全量同步后的完整校验

- **增量数据比对**：支持按时间字段（如创建时间、更新时间）、批次号、主键范围筛选，仅比对增量数据，适用于日常定时抽取、增量同步后的快速校验

- **增量字段配置**：支持为每张表单独配置增量时间字段名称、批次号字段名称，支持源表和目标表字段名不同时的映射配置

### 4.3.2 数据比对项

- 数据总行数比对：先快速校验源表、目标表总数据行数，行数不一致直接标记差异，无需逐行比对（快速筛查）

- 主键一致性比对：以表主键为唯一标识，校验源表主键在目标表是否存在，目标表是否有多余主键

- 字段值一致性比对：对主键匹配的行，逐字段比对数据值，支持数值、字符、日期、布尔等各类数据类型精准比对

- 空值比对：精准识别源库为NULL、目标库非NULL，或源库非NULL、目标库为NULL的差异

- 数据类型映射比对：自动处理不同数据库之间的数据类型差异（如Oracle的NUMBER对应MySQL的DECIMAL），内置常用数据库类型映射规则

- 浮点数精度比对：支持配置浮点数比对的精度容差（默认小数点后6位），在容差范围内视为一致

- 字符串比对规则：支持配置是否忽略前后空格、是否区分大小写、是否忽略全角/半角差异

- 日期时间比对：支持配置日期时间比对精度（精确到秒/毫秒），处理时区差异

### 4.3.3 数据差异处理

- 分类标记数据差异：行数不一致、主键缺失、主键多余、字段值不一致、空值差异

- 精准定位差异行：标注差异行的主键值、差异字段、源库值、目标库值

- 支持差异数据限量展示：避免大批量差异导致页面卡顿，默认展示前100条差异明细，支持查看全部

- 支持跳过大字段比对：针对文本、图片等大字段（CLOB、BLOB、TEXT等），可配置是否比对，提升比对速度

- 支持大表分页比对：对于数据量超过配置阈值（默认100万行）的大表，自动采用分页比对策略，避免内存溢出

## 4.4 比对结果管理与导出

- 实时展示比对进度：显示比对表数量、已完成表、当前比对表、比对耗时、进度百分比

- 汇总比对结果：展示总表数、结构一致表数、结构差异表数、数据一致表数、数据差异表数、无差异表总数

- 支持结果导出：导出标准化比对报告，格式支持Excel（测试人员常用）、HTML、TXT，报告包含数据源信息、比对时间、比对范围、结构差异明细、数据差异明细、汇总结论

- 支持结果保存：工具本地保存历史比对记录，支持按时间、业务场景、表名查询历史结果，方便追溯

- 历史记录管理：支持配置历史记录保留策略（默认保留最近90天或最多500条记录），支持手动清理历史记录释放存储空间

- 支持结果对比：可将本次比对结果与历史比对结果进行对比，快速识别新增/消除的差异

## 4.5 日志与异常处理

- 比对日志记录：全程记录比对过程，包括连接日志、表加载日志、比对日志、异常日志，方便排查问题

- 异常提示与处理：数据库连接中断、无权限访问表、表不存在、数据类型不兼容等异常，实时弹出友好提示，说明异常原因和解决建议

- 支持断点续比：比对过程中出现异常中断，下次可从断点处继续比对，无需重新开始，断点信息本地保存有效期为7天

- 支持任务管理：可同时创建多个比对任务，支持任务排队执行，支持查看任务执行状态和进度

## 4.6 基础设置功能

- 支持比对线程配置：根据电脑性能调整比对线程数，提升比对速度

- 支持超时时间配置：自定义数据库查询超时、比对超时时间

- 支持差异忽略规则配置：可设置忽略某些特定字段、特定类型的微小差异（如业务允许的精度误差）

- 忽略规则详细配置：
  - 按字段名忽略：指定忽略比对的字段名（支持通配符，如*_TIME、UPDATE_*）
  - 按数据类型忽略：指定忽略比对的数据类型（如TIMESTAMP、BLOB）
  - 按差异类型忽略：指定忽略的差异类型（如仅空格差异、大小写差异）
  - 按表名配置：不同表可配置不同的忽略规则

- 支持比对模板：可将常用的比对配置（数据源、表范围、忽略规则等）保存为模板，便于复用

# 5. 非功能需求

## 5.1 准确性需求

工具比对准确率需达到100%，无漏判、无错判，所有结构和数据差异均能精准识别，比对结果与人工逐表逐行核对结果完全一致。

## 5.2 性能需求

- 单表结构比对：100张表以内结构比对耗时不超过1分钟

- 数据比对：单表10万行数据全量比对耗时不超过5分钟，增量比对耗时不超过1分钟

- 大批量比对：50张表、单表平均50万行数据，全量比对耗时不超过30分钟

- 工具运行内存占用合理，无卡顿、无崩溃，支持Windows10及以上系统流畅运行

## 5.3 易用性需求

- 界面简洁清晰，布局符合测试人员操作习惯，无需专业数据库知识即可上手

- 配置步骤简化，核心比对流程：选择数据源→选择比对表→启动比对→查看结果→导出报告，三步完成核心操作

- 所有功能按钮、提示文字通俗易懂，无专业术语堆砌，异常提示附带解决方法

## 5.4 兼容性需求

支持Windows10、Windows11、macOS 12及以上桌面系统，适配主流数据库版本：

| 数据库类型 | 支持版本 |
|-----------|----------|
| MySQL | 5.7、8.0及以上 |
| Oracle | 11g、12c、19c及以上 |
| SQL Server | 2016及以上 |
| PostgreSQL | 10及以上 |
| 达梦（DM） | DM7、DM8及以上 |
| Inceptor | 6.0及以上 |

无需额外安装数据库客户端，工具内置各数据库JDBC驱动。

## 5.5 安全性需求

- 数据库密码加密存储，本地仅保存加密后内容，禁止明文传输和展示

- 工具仅做只读查询操作，禁止对源库、目标库进行新增、修改、删除等写操作，保障数据安全

- 工具仅限内部使用，无外网传输数据功能，所有比对结果仅本地存储

## 5.6 可维护性需求

支持后续数据库类型扩展、功能迭代优化，日志清晰便于后期问题排查，代码结构规范便于维护。

# 6. 界面与交互需求

- 采用桌面端可视化界面，分四大模块：数据源配置、比对任务设置、比对结果展示、历史记录

- 数据源配置区：表单式录入，一键测试连接，清晰展示已保存数据源列表

- 比对任务区：单选全库/指定表，选择全量/增量模式，一键启动/暂停/停止比对

- 结果展示区：顶部汇总统计卡片，中间结构差异、数据差异分标签页展示，底部明细列表

- 操作按钮：导出报告、清空结果、重新比对、查看日志等常用按钮放在显眼位置

# 7. 部署与环境需求

## 7.1 运行环境

- 操作系统：Windows10/11 64位、macOS 12及以上

- 硬件配置：
  - 最低配置：CPU i5及以上，内存8G及以上，硬盘剩余空间10G及以上
  - 推荐配置：CPU i7及以上，内存16G及以上，硬盘剩余空间50G及以上（大批量数据比对场景）

- 依赖环境：内置运行时环境，无需用户额外安装

- 网络要求：运行电脑需能连通源数据库和目标数据库服务器

## 7.2 部署方式

- 绿色版桌面工具，无需安装：
  - Windows版本：解压后直接运行.exe文件
  - macOS版本：解压后直接运行.app文件

- 支持拷贝至任意电脑使用，配置文件本地存储

- 首次运行自动初始化本地数据库（SQLite），用于存储配置和历史记录

- 支持配置导入/导出：可将数据源配置、比对模板等导出为加密文件，便于在不同电脑间迁移

# 8. 验收标准

1. 完成所有核心功能开发，数据源配置、结构比对、数据比对、结果导出功能均可正常使用

2. 测试人员模拟实际ETL业务场景，选取20张不同类型表进行全量比对，差异识别准确率100%

3. 性能指标满足非功能需求，大批量表比对无崩溃、无卡顿

4. 导出的Excel报告格式规范，明细完整，可直接用于测试存档

5. 工具操作流畅，核心测试人员可在10分钟内学会使用，无操作障碍

6. 工具仅做只读操作，未对源、目标数据库产生任何数据修改，数据安全可控

# 9. 风险与备注

## 9.1 风险说明

| 风险类型 | 风险描述 | 应对措施 |
|---------|---------|----------|
| 数据库权限风险 | 需用户提前申请源库、目标库的只读查询权限，否则无法完成比对 | 工具启动时提供权限检查功能，提前发现权限问题 |
| 网络风险 | 工具运行电脑需能连通源库和目标库，网络波动会影响比对速度 | 支持断点续比，网络恢复后可继续执行 |
| 特殊数据类型风险 | 数据库自定义数据类型、空间数据类型可能无法正确比对 | 后续版本根据业务需求迭代优化 |
| 大数据量风险 | 超大表比对可能导致内存占用过高或比对时间过长 | 采用分页比对策略，支持配置资源使用上限 |
| 数据安全风险 | 比对结果可能包含敏感业务数据 | 报告加密存储，支持配置敏感字段脱敏展示 |

## 9.2 使用限制

- 本工具仅用于ETL抽取后的一致性校验，不可用于数据库数据修改、同步等其他操作

- 单次比对任务建议表数量不超过500张，超大规模比对建议分批执行

- 工具仅限公司内部使用，禁止对外传播
