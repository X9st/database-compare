# 数据库比对工具需求功能文档

## 1. 项目概述

### 1.1 项目背景
为测试人员提供一款本地桌面应用，自动化比对不同数据库的表结构、字段信息和数据内容，替代人工肉眼比对，提高测试效率和准确性。

### 1.2 技术选型
- **框架**: Electron（跨平台桌面应用）
- **前端**: React/Vue + TypeScript（支持离线部署，所有依赖打包到本地）
- **后端**: Python 3.7（通过 Electron 的 `python-shell` 或子进程调用 Python 服务）
- **通信方式**: Electron 主进程通过 HTTP/IPC 与 Python 后端服务通信
- **Python 依赖**: 使用 `requirements.txt` 管理，支持离线安装（`pip install --no-index --find-links`）
- **数据库驱动**: 支持多种数据库连接（见 2.1.1）

### 1.3 Python 后端技术栈
- **Web 框架**: Flask 或 FastAPI（轻量，适合桌面应用集成）
- **数据库连接**:
  - MySQL: `PyMySQL` / `mysql-connector-python`
  - PostgreSQL: `psycopg2`
  - SQL Server: `pyodbc` / `pymssql`
  - Oracle: `cx_Oracle`
  - SQLite: 内置 `sqlite3`
  - 达梦: `dmPython`（达梦官方驱动）
  - Inceptor: `pyhive` / `impyla` / JDBC
  - 人大金仓: `ksycopg2`
- **数据处理**: `pandas`（数据比对）、`numpy`
- **报告生成**: `openpyxl`（Excel）、`jinja2`（HTML 模板）
- **配置管理**: `pydantic`（数据验证）、`pyyaml`

## 2. 功能需求

### 2.1 数据库连接管理

#### 2.1.1 数据源配置
- **功能描述**: 支持配置多个数据库连接（源数据库和目标数据库）
- **支持数据库类型**:
  - **国际数据库**: MySQL / MariaDB、PostgreSQL、SQL Server、Oracle、SQLite、MongoDB（可选扩展）
  - **国产数据库**: 
    - 达梦数据库 (DM Database) - 使用 `dmPython` 驱动
    - Inceptor (星环信息) - 使用 JDBC/ODBC 或 `pyhive` 驱动
    - 人大金仓 (Kingbase) - 使用 `ksycopg2` 驱动（兼容 PostgreSQL）
    - 南大通用 (GBase) - 使用对应 Python 驱动
    - 神舟通用 (Oscar) - 使用对应 Python 驱动
- **连接信息**:
  - 连接名称（自定义标识）
  - 数据库类型
  - 主机地址 + 端口
  - 数据库名/实例名
  - 用户名 + 密码
  - 连接参数（可选，如字符集、SSL等）
- **连接测试**: 保存前可测试连接是否成功
- **连接管理**: 保存常用连接配置，支持增删改查

#### 2.1.2 连接安全
- 密码本地加密存储（使用系统密钥链或加密文件）
- 支持连接池管理
- 连接超时设置

---

### 2.2 比对任务配置

#### 2.2.1 数据源选择
- 从已保存的连接中选择**源数据库**和**目标数据库**
- 支持同一类型数据库比对，也支持跨类型数据库比对（如 MySQL vs PostgreSQL）

#### 2.2.2 比对范围选择
- **全库比对**: 比对两个数据库中所有表
- **指定表比对**: 手动选择需要比对的表
- **表名映射**: 支持源表和目标表名称不同的比对（如 table_a vs table_a_new）

#### 2.2.3 比对维度配置
用户可勾选需要比对的维度：

| 维度 | 比对内容 |
|------|----------|
| **表结构** | 表名、表注释、存储引擎、字符集等 |
| **字段信息** | 字段名、字段类型、长度/精度、是否可空、默认值、字段注释、主键、索引等 |
| **数据内容** | 记录数量、数据一致性（抽样或全量）、数据差异行 |

#### 2.2.4 数据比对高级选项
- **抽样比对**: 设置抽样比例或抽样数量，提高大数据量比对的效率
- **条件过滤**: 支持 WHERE 条件过滤需要比对的数据
- **忽略字段**: 可指定不参与比对的字段（如自增 ID、更新时间等）
- **排序规则**: 指定数据比对的排序字段

---

### 2.3 三维度比对功能

#### 2.3.1 表结构比对
- 检测源库存在但目标库不存在的表（新增表）
- 检测目标库存在但源库不存在的表（删除表）
- 比对表的元数据差异（引擎、字符集、注释等）

#### 2.3.2 字段信息比对
- 检测字段的新增、删除、修改
- 字段属性差异检测：
  - 字段类型变更（如 INT → VARCHAR）
  - 长度/精度变更（如 VARCHAR(50) → VARCHAR(100)）
  - 可空性变更（NULL → NOT NULL）
  - 默认值变更
  - 注释变更
- 索引差异检测（新增、删除、修改索引）
- 主键差异检测

#### 2.3.3 数据内容比对
- 记录数量比对
- 数据一致性比对：
  - 逐行比对（适合小数据量）
  - 哈希比对（适合大数据量）
  - 抽样比对
- 差异数据识别：
  - 源库存在但目标库不存在的记录
  - 目标库存在但源库不存在的记录
  - 主键相同但内容不同的记录

---

### 2.4 比对结果展示

#### 2.4.1 实时比对进度
- 显示当前比对阶段（连接数据库 → 获取元数据 → 比对表结构 → 比对字段 → 比对数据）
- 进度条显示整体进度
- 可取消正在进行的比对任务

#### 2.4.2 差异结果展示
- **概览面板**: 统计差异数量（新增/删除/修改的表数量、字段数量、数据差异行数）
- **差异列表**: 
  - 树形结构展示：数据库 → 表 → 字段/数据
  - 差异项用颜色标识：
    - 🟢 绿色：新增
    - 🔴 红色：删除
    - 🟡 黄色：修改
- **详情对比**: 点击差异项可查看源库和目标库的详细对比

#### 2.4.3 数据差异展示
- 表格形式展示差异数据
- 差异字段高亮显示
- 支持分页浏览
- 支持搜索和筛选

---

### 2.5 报告导出

#### 2.5.1 导出格式
- **HTML 报告**: 美观的网页格式，可在浏览器中打开
- **Excel 报告**: 便于进一步处理和分享
- **PDF 报告**: 适合归档和打印
- **JSON/XML**: 便于程序化处理

#### 2.5.2 报告内容
- 比对任务基本信息（时间、数据源、比对维度）
- 差异汇总统计
- 详细的差异清单
- 数据差异样例（前 N 条）

### 2.6 历史记录与任务管理

#### 2.6.1 比对历史
- 自动保存比对任务配置和结果
- 支持查看历史比对记录
- 支持重新执行历史任务

#### 2.6.2 任务模板
- 保存常用的比对配置为模板
- 快速创建比对任务

## 3. 离线部署方案

### 3.0.1 前端离线部署
- **静态资源打包**: 使用 Electron-builder 将前端资源打包到应用内
- **无 CDN 依赖**: 所有 UI 组件库（如 Ant Design、Element Plus）通过 npm 安装，打包时包含在应用内
- **无外部 API 调用**: 应用不依赖任何外部网络服务
- **离线安装包**: 提供完整的离线安装包（.exe/.dmg/.AppImage）

### 3.0.2 Python 环境离线部署
- **Python 解释器打包**: 使用 `pyinstaller` 将 Python 后端打包为可执行文件，或携带便携版 Python
- **依赖包离线安装**:
  - 预先下载所有依赖包到 `dependencies/` 目录
  - 安装脚本自动执行 `pip install --no-index --find-links=dependencies -r requirements.txt`
- **数据库驱动离线安装**: 国产数据库驱动（如 dmPython）需提前下载 whl 文件

### 3.0.3 部署目录结构
```
DatabaseCompareTool/
├── app/                          # Electron 应用
│   ├── main/                     # 主进程
│   ├── renderer/                 # 渲染进程（前端）
│   └── python-service/           # Python 后端服务
│       ├── python/               # 便携版 Python 3.7（或 pyinstaller 输出）
│       ├── requirements.txt
│       ├── dependencies/         # 离线依赖包
│       └── src/                  # Python 源码
├── resources/                    # 静态资源
└── install-scripts/              # 安装脚本
```

## 3. 非功能需求

### 3.1 性能要求
- 支持大数据量比对（百万级记录）
- 比对过程不阻塞 UI（使用多线程/子进程）
- 支持分批处理和流式处理

### 3.2 用户体验
- 直观的图形化界面
- 清晰的错误提示
- 操作引导和帮助文档
- 支持深色/浅色主题

### 3.3 安全性
- 数据库密码加密存储
- 敏感数据（如密码字段）在展示时脱敏
- 本地运行，数据不上传云端

### 3.4 兼容性
- 支持 Windows、macOS、Linux 系统
- 支持常见数据库版本
